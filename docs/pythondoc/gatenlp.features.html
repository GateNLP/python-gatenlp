gatenlp.features module¶

class gatenlp.features.Features(*args, logger=None, **kwargs)[source]¶

Bases: collections.UserDict

A dict-like class for storing features, which are mappings from string feature names to arbitrary feature values. If the Features instance is a field in another object where changes are getting logged in a change log, it should pass on the logger, a method for logging feature changes. Any copy of an instance of Features will not receive the logger, in order to make sure that logging happens, the instance stored in the original owning object must be used.

Initialize a Features object.

Parameters

  • initialfeatures – the initial features, as for a dict.

  • logger – a function for logging any changes to the feature map. This should be a method implemented in the owning object. It should take the following parameters: command, featurename, featurevalue.

clear()[source]¶

Remove all features.

Returns

copy(deep=False)[source]¶

Return a shallow (or deep if deep=True) copy of the features. The result is another instance of Features which is detached from the owner and which does not log the changes. However, if the copy is shallow and feature values are references to mutable objects, they can still get modified in the original set (without any logging!).

Parameters

deep – if True return a deep instead of a shallow copy of the features.

Returns

a dictionary with the features

static from_dict(thedict, copy=True, deepcopy=False)[source]¶

Create a Features instance from a dictionary. If copy is True, a shallow copy of the dictionary is used, if deepcopy is True as well, a deepcopy is created instead.

NOTE: no checks are done to make sure that feature names are string only!

Parameters

  • thedict – the dictionary from which to create the Features.

  • copy – if True use a shallow copy of the dictionary

  • deepcopy – if True and copy is True, use a deep copy of the dictionary

Returns

the Features instance

to_dict(copy=True, deepcopy=False)[source]¶

Return a dictionary representation of the features.

Parameters

  • copy – if True, the dictionary is a shallow copy of the dictionary wrapped in the Features object. This should always be done except it is known that no modifications are made to the dictionary or the modifications do not matter in the original Features object.

  • deepcopy – if True and copy is True, the dictionary is a deep copy so that mutable objects in the original are unaffected if they get modified in the copy.

Returns

the dict
